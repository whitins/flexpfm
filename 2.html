2.서비스개요

1) 조직 목표 관리
· 조직목표 : 조직목표 내용, 완료예정일, 달성도 실적을 등록함.
· 과제관리 : 설정한 목표를 달성하기 위한 과제 정보를 등록하고, 과제를 수행할 팀원을 지정함. 과제를 할당받은 직원은 본인의 업무 계획 수립에서 연동되고, 조직목표와 개인 업무계획이 정렬되도록 함.

2) 개인 성과 관리 기능 개발
· 개인 업무 등록 : 업무 등록(조직 과제 연계 여부 선택), 업무 수행기간, 목표 수준, 상세 수행방안, Co-Worker 정보를 등록하여 성과관리자(리더)에 제출.
· 업무 승인 : 제출된 업무 정보를 검토하여, 승인/반려 처리
· 수시 기록 : 업무 진행상황을 공유을 위한 기능으로, 커뮤니케이션 기능 강화
· 실적 등록 : 업무가 완료된 시점에 실적을 등록(체크인)하고, 제출 함.
· 마이체크인 : 본인 업무 Dash Board, 진행현황, 체크인, 수시기록, 동료코멘트
· 피드백 : 제출된 업무 실적 내용을 검토하고, 평가결과와 의견을 입력함.
· 확정/이의제기 : 피드백 결과를 조회하고, 결과에 대해 수락 /이의제기를 함.

3) 조직 성과 관리 기능 개발
· 팀업무보드 : 리더의 팀 업무 Board, 부서장(리더)를 위한 Dash-Board 기능으로 팀원이 현재 진행 중인 업무의 상세내용과 진행상황을 파악할 수 있음. 개별 메뉴로 제공되는 피드백 결과를 Dash-Board 화면에서도 수행할 수 있도록 하여 사용자 편의성을 제고하도록 설계 됨.
· 팀 주간 업무 : 현재 주에 완료 계획으로 된 업무와 계획 대비 지연되고 있는 업무가 함께 조회되어, 미 완료된 업무의 진행상황을 공유할 수 있음. 이로 인해, 향후 평가결과에 대한 부서원의 수용도를 높여서, 평가결과에 대한 신뢰도를 제고하는 기여 함.
· 팀 업무 이력 : 피드백이 완료된 업무에 대한 현황을 제공하여, 본인의 평가성향을 파악하여 평가Bias (평가 관대화, 가혹화)가 발생되지 않도록 예방하고, 팀 전체의 성과를 누적으로 모니터링 할 수 있음.
· 성과관리 현황 : 주로 팀 이상의 상위 관리자에게 하위 팀단위의 업무 진행현황을 쉽게 파악하도록, 조직 구조 Tree 형태로 정보를 가공하여 제공 함. 팀 단위의 업무 진행현황 뿐만 아니라, 하위 부서장의 평가성향 파악함으로써 조직 관리에 필요한 정보를 제공 함.

4) 역량 키워드 기반의 동료/상사 다면평가 기능 개발 · 평가지표 및 평가문항 관리 : 평가지표 정보를 등록하고, 직군 및 평가군 별로 평가문항을 설정하여, 회사 직군에 맞는 평가지표를 설정할 수 있음. 이러한 기능은 시스템 관리자가 아니라, 평가 담당자가 직접 설정할 수 있도록 하여, 사용자의 요구에 따라 다양한 케이스를 관리할 수 있음.
· 다면평가자 관리 : 개인 업무에 등록된 Co-worker 데이터를 활용하여 다면평가자가 기본 지정되도록 하여 해당 직원 함께 협업한 동료가 평가에 참여하게 하여 평가결과에 대한 정확성을 확보할 수 있음. 팀 인원, Co-worker 적은 경우 충분한 인원수의 평가자를 얻을 수 없는 경우, 부서장이 추가로 조정할 수 있도록 설계되어 잇음.
· 다면평가 참여 : 앞의 다면평가자 지정을 통해 본인이 평가해야 할 대상자를 확인하고 다면평가자가 평가가 어렵다고 판단하는 경우, 평가참여에 포기할 수 있는 기능을 제공 함. 평가항목은 평가지표 내용을 쉽게 파악할 수 있도록 질문지 방식으로 제공할 수 있음.
· 다면평가 결과 집계: 다면평가 결과 집계하여 평가영역/항목별 빈도 분석하여, 본인의 강점 영역과 개선할 영역을 제시하여 피드백 됨. 아울러 다면평가자 들이 입력한 코멘트도 익명처리하여 조회 되도록 하여 동료/상사의 피드백 기능을 제고할 수 있음.
